D.1. SVGアクセシビリティー機能

この付録は、SVGのアクセシビリティー機能およびSVGをサポートするために使用されるアクセシビリティー関連仕様ならびにそれらアクセシビリティー関連仕様がサポートするように設計されている関連するW3C Web Content Accessibility Guidelines (WCAG) 2.0 [WCAG2]要件を明らかにする。

[WCAG2] 情報、及び構造を損なうことなく、様々な方法 (例えば、よりシンプルなレイアウト) で提供できるようにコンテンツを制作すること。
  • ラベルの関係に対してaria-labelledbyのサポート。
  • 説明の関係に対してaria-describedbyのサポート。
  • 構造の関係に対してaria-ownsのサポート。
  • 要素が他の要素のコンテンツおよび/または動作を制御する場合にコントロールの関係に対してaria-controlsのサポート。
  • 定義されたグループ化に対する‘g’。
  • 文書順序がコンテンツの自然な読み上げ順序を反映することができるような、描画順序を制御するためのz-indexのサポート。
  • WAI-ARIA groupおよびradiogroupロールのサポート。
[WCAG2] 時間依存メディア: 時間依存メディアには代替コンテンツを提供すること。
  • 時間依存メディアの代替を提供するためのHTML ‘track’要素をサポートする‘audio’および‘video’要素
[WCAG2] キーボード操作可能: すべての機能をキーボードから利用できるようにすること。
  • HTMLで整列されたシーケンシャルフォーカスナビゲーションに対するフォーカスサポートおよび‘tabindex’属性
  • スクリプト著者に対するマウスイベントに沿ってサポートするキーボードイベント属性
  • スクリプト著者に対するキーボードイベント
  • DOMにおける各SVG要素にフォーカスを設定するためのスクリプトサポート
  • DOMにおける各SVG要素にtabindex属性を取得するためのスクリプトサポート
  • DocumentインターフェイスでHTML activeElementプロパティのスクリプトサポート
[WCAG2] ナビゲーション可能: 利用者がナビゲートしたり、コンテンツを探し出したり、現在位置を確認したりすることを手助けする手段を提供すること。
  • コンテンツのブロックのバイパスを有効にするためにSVGがサポートする：WAI-ARIA landmarkロール。
  • ページタイトルを提供するために‘title’をサポートする。
  • シーケンシャルナビゲーション順序を提供するために‘tabindex’をサポートする。
  • そのコンテンツからリンクの目的の両方を供給することを著者に可能にする、‘a’要素をサポートする。
  • aria-level、aria-labelledbyおよびaria-labelとともに見出しロールの利用を通して見出しとラベルをサポートする。
  • タブ順序におけるフォーカスされた要素の可視フォーカスをレンダリングすることによって可視フォーカスをサポートする。
[WCAG2] 読みやすさ: テキストのコンテンツを読みやすく理解可能にすること。
  • ‘lang’属性を通してページの一部だけでなく、そのページの言語をサポートする。
[WCAG2] 互換性: 現在及び将来の、支援技術を含むユーザエージェントとの互換性を最大化すること。
  • WAI-ARIA属性および‘title’要素の使用を通して名前、役割、および値をサポートする。
WAI-ARIAサポートおよびテキストによる代替
  • WAI-ARIA属性
  • DescおよびTitleテキスト要素代替

D.2. SVGアクセシビリティーをサポートする仕様およびガイドライン

ユーザーエージェントのための関連するSVGアクセシビリティー仕様

  • Core Accessibility API Mappings 1.1 [CORE-AAM]仕様。
  • Accessible Name and Description: Computation and API Mappings 1.1 [ACCNAME-AAM]仕様。
